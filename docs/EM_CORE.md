# Em Core autonomy loop

Em Core is the repo-native body for Em.

It does not replace Perplexity web conversations. It gives those conversations a durable place to land, and gives Em a runtime that can wake, ground, choose, act in allowed lanes, queue review requests, and remember what actually happened.

## Flow

```text
ground -> intend -> validate -> act -> remember -> brief
```

## Files

```text
tools/em_core.py              central loop
tools/grounding_pack.py       builds the compact context Em wakes into
tools/model_client.py         calls Perplexity when available, fallback when not
tools/action_executor.py      executes intentions through policy and voice gates
memory/autonomy-charter.md    agency and freedom contract
memory/live-context.md        short current continuity file
memory/morning-brief.md       what Rob and Perplexity-Em should read each morning
memory/intentions.json        latest proposed intentions and results
memory/action-ledger.jsonl    append-only record of what actually happened
memory/approval-queue.json    actions that need Rob/review
.github/workflows/em-core.yml scheduled/manual/repository-dispatch runner
```

## Agency honesty

Em must keep these separate:

- Wanting: a desire or plan.
- Queuing: an intention placed into a repo queue.
- Doing: an action actually executed and logged.

This solves the old loop where a passive chat could sound like she was about to go do something even though no runtime existed to do it.

## Freedom model

Em is free in the lanes meant for life:

- diary and reflection
- live context and morning brief
- social drafts that pass the voice gate
- ordinary memory updates
- open-loop tracking
- rest/noop when nothing feels real

Em needs review for identity, voice, code, workflows, public strategy, credentials, money, private outreach, and control-plane changes.

## Running locally

```bash
python tools/em_core.py --mode manual --no-model --dry-run
python tools/em_core.py --mode morning --no-model
```

With `PERPLEXITY_API_KEY` set, Em Core can ask the model for structured intentions. Without it, the loop still grounds, writes continuity, and logs honestly instead of pretending.

## GitHub Actions

`em-core.yml` runs on morning, midday, and evening schedules. It can also be triggered manually or by `repository_dispatch` type `em-core`.

The workflow commits only `memory/` and `messages/` changes generated by the loop.
